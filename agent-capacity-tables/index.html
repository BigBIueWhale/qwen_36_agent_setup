Definitions & assumptions

Hot users
Full‑context conversations whose KV cache is resident in GPU memory (HBM/GDDR7) and can generate tokens right now.
Live users
Hot users plus conversations parked in system RAM via KV‑cache offload, paged back into GPU memory when their turn runs. Assumes the serving engine (vLLM) implements KV offload for hybrid‑attention models.
8‑bit weights / full‑quality KV
Model weights quantized to FP8 (~1 byte/parameter). KV cache kept at 16‑bit BF16 (2 bytes/element) — no KV quantization anywhere.
Context length
256K = 262,144 tokens; 1M = 1,048,576 tokens. Ornith‑1.0‑35B‑A3B and Ornith‑1.0‑397B run at 1M; Google Gemma 4 31B runs at its 256K native context (it does not support 1M). Gemma's figure assumes vLLM exploits its 1,024‑token sliding‑window cap on 50 of 60 layers.
Pricing
